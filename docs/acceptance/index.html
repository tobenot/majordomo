majordomo · 指挥官 验收 Guide

v0.2 · Claude Code 常驻 SDK 会话 · 第三代 AI 工作流脚手架

一句话结论：整条链路已跑通且消除了伪功能 —— SDK 工作层现在是常驻 streaming-input 会话，/compact、/model 和真实权限确认都走同一底层 session。 验收管道仍可强制 mock；真实工作层默认 permissionMode: "auto"。

1构建

在项目根目录执行（已装好依赖；若没装先 npm install）：

npm run build
npm run doctor
npm run selftest

产物在 dist/，Web 静态资源会自动拷到 dist/web/public。

  • 构建无报错，出现 [copy-assets] web public -> dist/web/public
  • doctor 打印 Node / SDK / profile 命令 / Web 资源 / 通知脚本状态
  • selftest 输出 SELFTEST OK，事件链包含 permission_request 和 persona_message

2验收 TUI（开箱即跑，强烈建议先用 mock）

auto 会优先尝试真实 Agent SDK。若只是验收 UI / 协议管道，请先强制 mock：复制 config.example.jsonc 为 config.jsonc，把 worker.engine 改成 "mock"。 
# 复制配置并改 engine 为 mock（一次性）
copy config.example.jsonc config.jsonc
# 然后编辑 config.jsonc，把 "engine": "auto" 改成 "engine": "mock"

# 启动 TUI（默认命令即 attach，内嵌 daemon + 进入 TUI）
node dist/cli.js

进入后直接打字回车（无会话会自动新建），观察：

  • 顶部出现 已连接 · profile=home · 工作层=mock
  • 输入「帮我看看代码」→ 看到灰色 工作层 流式输出
  • 随后看到紫色 指挥官 用猫娘口吻的人话汇报
  • 输入含「删除」或「rm」→ 弹出黄色权限请求，问 y/n
  • 回 y 批准 → 工作层继续；回 n 拒绝 → 跳过

TUI 命令

命令作用
/new [名字]新建会话
/sessions 或 /ls列出所有会话（报菜名）
/resume <id>续接历史会话
/profile <名>切换 profile（只影响新开会话）
/compact透传给常驻 SDK session，手动压缩上下文
/model <模型>透传给常驻 SDK session，切换后续模型
/quit退出

3验收 Web 面板（看立绘位 / 会话管理）

node dist/cli.js web

浏览器打开提示的地址（默认 http://127.0.0.1:4318）：

  • 右上角连接状态变绿「已连接」，显示工作层引擎与 profile 下拉
  • 左侧会话列表（与 TUI 同一份状态，TUI 里建的会话这里能看到）
  • 左下角有「立绘 / CG」占位框（后续迭代填真图）
  • 输入框发消息 → 我方蓝气泡、工作层灰气泡、指挥官紫气泡分明
  • 触发权限 → 顶部黄条出现「批准 / 拒绝」按钮
  • 切换 profile 下拉 → 提示「已切换（只影响新开会话）」
  • 访问 /healthz 返回 JSON：{ ok, web, assets, daemonWs }，且不暴露本机绝对路径

4验收 Profile 机制 & 配置

# 查看解析后的配置与来源
node dist/cli.js config

# 切换活跃 profile（持久化到 ~/.majordomo/config.jsonc）
node dist/cli.js profile internal
  • config 打印出 home / internal / tclaude 三个 profile
  • profile internal 后再 config，activeProfile 变为 internal
  • 确认 internal 的 personalDir 是 ~/.claude-internal（不是 .claude，这是关键坑）

5接真实工作层（可选，需已登录 claude）

把 config.jsonc 的 worker.engine 设回 "auto" 或 "sdk"。"cli" 只作为 SDK 不可用时的基础兜底。

正式 SDK 包名是 @anthropic-ai/claude-agent-sdk。SDK Worker 使用 streaming input 常驻会话，连续性在内存里保持； 捕获到的 session_id 仍会持久化，主要用于崩溃后的 resume 兜底。 permissionMode: "auto" 保持原样交给 Claude Code 分类器；需要人工介入时由 canUseTool 回调转给 TUI / Web。 
# config.jsonc
"permissionMode": "auto",
"worker": { "engine": "sdk" }
  • doctor 显示 SDK package : import ok，Resolved engine 为 sdk
  • 会话 engine 显示为 sdk
  • 真实回答经人设层润色后汇报；底层 session_id 被存下（/sessions 可见）
  • 输入 /compact 后触发 SDK compact，并看到「上下文压缩」工作层消息
  • 触发需人工确认的工具时，TUI / Web 出现权限确认；批准/拒绝会回传给同一 SDK session
  • 可配置 maxTurns、timeoutMs、allowedTools、disallowedTools

6接真实人设层 API（可选）

复制 .env.example 为 .env，按接口类型选择 OpenAI-compatible 或 Anthropic Messages API：

# OpenAI-compatible /chat/completions
PERSONA_API_FORMAT=openai
PERSONA_API_KEY=你的key
PERSONA_API_BASE=https://你的网关/v1
PERSONA_MODEL=gpt-4o-mini

# Anthropic Messages API /v1/messages
PERSONA_API_FORMAT=anthropic
PERSONA_API_KEY=你的key
PERSONA_API_BASE=https://api.anthropic.com
PERSONA_MODEL=claude-3-5-haiku-latest
  • OpenAI-compatible 启动日志出现「检测到人设层 API 配置，使用 openai 模式」
  • Anthropic 启动日志出现「检测到人设层 API 配置，使用 anthropic 模式」；官方接口可省略 PERSONA_API_BASE
  • 汇报内容由真实模型生成（比离线模板更自然）；API 失败会自动降级转述，不会崩

7回归测试要点（改动后重点验）

  • 降级链路：删掉 config.jsonc 和 .env 后仍能跑（mock + 模板）。
  • 权限流程：mock 下输入危险词必弹权限；SDK 下高危工具应由 canUseTool 转发给前端确认。
  • 常驻连续性：SDK 同一会话内连续追问应保留上下文，/compact 后再追问仍能接上。
  • 多会话隔离：开两个会话交替输入，消息不串台（按 sessionId 路由）。
  • 状态一致：TUI 建的会话，Web 刷新后能看到（同一 ~/.majordomo/sessions.json）。
  • profile 持久化：切换后重启 daemon，activeProfile 保持。
  • 异常自洽：故意配错端口 / 错 profile，应给出清晰报错并降级，不静默崩溃。

!当前已知限制

  • CLI fallback 是一次性 headless 回合，只保证基础任务可运行；需要常驻上下文、/compact、交互权限时请使用 SDK worker。
  • 文档层（另开 session 写验收文档）、立绘/CG 渲染、远程接入（CF Access / 推手机）均已留好架构口子，尚未实现。
  • Windows 下已规避 Node shell+args 的 DEP0190 警告；如仍出现，优先看 doctor 输出。
