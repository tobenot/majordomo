majordomo 使用指南

装中枢 · 装插件 · 日常怎么用 —— 十分钟上手

0一句话它是什么

你手边同时开着好几个 Claude Code 窗口在干活。majordomo 不替你干活， 它站在旁边当管家：每个窗口做完一轮、或卡住等你许可，都会自动上报给中枢， 中枢维护三张表（谁在做什么 全局待办 待验收）， 再用人设口吻把技术输出翻成人话，你离场时还能推手机。

干活窗口 ×N
（装了 Bifrost 插件） → 中枢
三张表 + 人设复命 → 你
网页面板 / 手机

所以上手就两件事：把中枢跑起来，然后给每个干活窗口装上 Bifrost 插件。

1把中枢跑起来

在 majordomo 仓库目录里：

npm install
npm run build
npm run doctor      # 环境自检，绿了再往下
node dist/cli.js    # 起中枢 + Web 面板（默认命令）

看到这行就是中枢就绪：

core daemon 监听 http+ws://127.0.0.1:4350（上报入口 /ingest）

看面板

浏览器打开 http://127.0.0.1:4351（面板端口 = 中枢端口 + 1）。 现在还是空的——因为还没有窗口上报。右上角应显示 已连接。

端口备忘：中枢用 4350（WebSocket 和 /ingest 共用），面板 4351。 用 4350 是因为 4317 被企业微信占了。

想让它复命说人话（可选）

不配也能跑（离线模板会兜底出一句猫娘口吻）。想要真人设，复制 .env.example 为 .env 填 API：

PERSONA_API_FORMAT=openai        # 或 anthropic
PERSONA_API_KEY=sk-...
PERSONA_MODEL=gpt-4o-mini        # 便宜模型就够

填完重开中枢生效。用便宜小模型即可——它只负责把技术输出翻成人话，不干活。

2给每个干活窗口装 Bifrost 插件

这一步在你真正干活的 Claude Code 窗口做，不是中枢那个终端。启动时用 --plugin-dir 指到 bifrost/：

claude --plugin-dir D:/GitRep/majordomo/bifrost

把路径换成你机器上 majordomo 仓库的 bifrost 目录。然后照常用这个窗口干活即可。

装上后会发生什么

窗口发生的事中枢面板的反应
窗口启动左栏冒出这个窗口，状态 空闲
做完一轮回复persona 把这轮输出翻成一句人话复命；本机弹窗+提示音
建了任务 / 完成任务右栏「全局待办」自动 +1 / 自动划掉（不烧 LLM）
卡住等你许可窗口转 等你，进「待验收」，你离场则推手机
/exit 退出窗口变灰离线
三个常见坑：
① 插件只在窗口启动时加载 —— 必须开全新窗口，旧窗口不生效。
② 改了插件代码热载用 /reload-plugins；改了 plugin.json 要重开窗口。
③ 中枢没开也不影响干活 —— 插件会把上报落盘缓存，下次中枢开着时自动补送。

改上报地址等

编辑 bifrost/report.config.jsonc。默认指 http://127.0.0.1:4350/ingest， 中枢在别的机器/端口就改这里的 ingestUrl（要和中枢的 port 对上）。

3日常怎么用

  • 干活照旧：你还是在原生 Claude Code 窗口里正常敲。majordomo 全程旁观，不打扰。
  • 面板当仪表盘：晾着一个浏览器 tab（或 Edge app 浮窗）扫一眼——哪个窗口忙、哪个卡住等你、还有啥待办没清。
  • 离场靠手机：配了 Bark 就在你不在电脑前时把「XX 窗口卡住等你」推到手机（需在配置里加 bark 并加 "bark" 到 notifiers）。
  • 面板能手动操作的：加/勾/删待办、把待验收标「已处理」。这些会同步到所有连着的前端。
v1 是只读仪表盘：面板不能对窗口发指令——卡住的窗口你还是回那个窗口本人点许可。 中枢的价值是「不让任何窗口的卡点被你漏掉」。「从面板插话/直达窗口」是后续版本的事。

4不确定装好没？30 秒自测

不用开真窗口，中枢起着，另开一个终端手动灌一条上报：

curl -s -X POST http://127.0.0.1:4350/ingest \
  -H "Content-Type: application/json" \
  -d '{"windowId":"TEST-1","event":"stop","cwd":"D:/demo","payload":{"text":"测试：重构完成"}}'

返回 {"ok":true}，面板左栏立刻冒出一个 demo 窗口、点开有 persona 复命 —— 说明中枢这侧通了。 再开真窗口挂 Bifrost，能实时跟随就全通了。

更细的逐事件验收：docs/acceptance/2026-07-02-hub-v1-acceptance.html。命令行自检：npm run doctor / npm run selftest。

5出问题先看这里

现象处置
curl 连接被拒中枢没起，或端口不对。确认 config.jsonc 的 port 和 bifrost/report.config.jsonc 的 ingestUrl 都指 4350。
面板空白但 curl 返回 ok浏览器要开面板端口 4351（不是 4350），且右上角显示已连接。
真窗口没反应Bifrost 只在窗口启动时加载 —— 必须开全新窗口。
复命很久才出同窗口有节流（默认 15s），高频 Stop 会跳过。想每条都出，调小 hub.personaThrottleMs。
中文乱码脚本读 stdin 前需 UTF-8（report.ps1 已内置）。若自改脚本记得设。
一开中枢涌进一堆旧窗口正常，是离线缓存补送。陈旧事件（>5min）只录表不复命不推送。不想要就清空 bifrost/cache/ingest.offline.jsonl。

6放服务器上？先加鉴权

⚠️ v1 只在 localhost 无鉴权。中枢一旦暴露到公网，/ingest 必须加 token（或放 Cloudflare Access 之类后面）， 否则任何人都能往你的待办灌数据。本地自用无碍，公网前务必补上。

另外服务器无桌面时本机弹窗/提示音全废，触达你的唯一出口是 Bark —— 记得配好手机推送。
